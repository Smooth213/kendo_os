# Emergency Recovery Procedure (緊急復旧手順)

自動判定が困難な事態や、深刻なデータ不整合が発生した際の人道的救済手順です。

## 1. Human Override (裁定の上書き)
システムが「一本」と判定しなくても、審判員が「一本」と認めた場合、オペレーターはエンジンの計算を無視して手動でスコアを加算できる権限を持ちます。

## 2. Replay-based Correction (履歴からの訂正)
誤入力に気づいた場合、DBを直接書き換えるのではなく、`Undo` イベントを発行して「歴史を巻き戻してやり直す」ことを原則とします。これにより、監査ログとの一貫性を保ちます。

## 3. Disaster Recovery (最終手段)
アプリが完全にフリーズし、リプレイによる復旧も不可能な場合は、以下の手順を実施します。
1. 現在のスコアと残り時間を紙の記録票に書き写す。
2. アプリをリセットし、新しく試合を作成。
3. 書き写した状態から手動で試合を再開させる（Continuity 優先）。
