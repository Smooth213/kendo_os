# Failure Policy Matrix (障害対応マトリクス)

システムコンポーネントごとの障害発生時の挙動方針（Fail-open / Fail-closed）を定義します。

| コンポーネント | 影響範囲 | ポリシー | 障害時の挙動 |
| :--- | :--- | :--- | :--- |
| **Rule Evaluation** | 判定・勝敗 | **Fail-closed** | 評価失敗時は操作をブロックし、審判主任の確認を促す。 |
| **Sync Service** | 端末間同期 | **Fail-open** | 同期に失敗してもローカル記録を続行し、 continuity を優先する。 |
| **Projection** | 掲示・モニター | **Fail-open** | 描画遅延や失敗を許容し、操作端末側の数値を「真」として続行する。 |
| **Audit Logging** | 監査ログ発行 | **Fail-open** | ログ記録に失敗しても試合操作は止めない。復旧後に整合性を確認する。 |

## ポリシー定義
- **Fail-open:** 障害が起きてもシステムの主要機能（試合記録）を継続させる。
- **Fail-closed:** データの整合性を守るため、安全が確認されるまで操作を停止させる。
