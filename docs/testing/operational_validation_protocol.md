# 現場運用検証プロトコル (Operational Validation Protocol)

本プロトコルは、Kendo Syncが実際の大会環境で「業務継続可能」であることを証明するための物理・運用テスト要件を定義する。
メジャーリリース前、または大規模なUI変更時には、必ず本プロトコルに基づく検証を実施し、結果を記録すること。

## 1. 体育館テスト (Gymnasium Environment Test)
剣道の試合会場特有の環境下で、マニュアルとシステムが機能するかを検証する。

- [ ] **視認性テスト (暗所・反射)**: 照明が暗い体育館内、または窓からの強い日差しがiPadの画面に反射する状況下で、UI（タイマー、得点）が明確に判別できるか。
- [ ] **遠距離テスト**: 本部席（または観客席）から、コートに置かれたiPadの表示（現在の試合状況）が視認できるか。
- [ ] **オフライン継続テスト**: 体育館の奥など、モバイルルーターや館内Wi-Fiの電波が完全に届かない場所で、部内戦モードやスコア入力が停止せずに動作し続けるか。

## 2. 高齢者・初見テスト (Elderly & First-Time User Test)
IT機器に不慣れな道場の先生や、急遽記録係を任された保護者を想定したテスト。

- [ ] **初見操作**: `docs/manuals/quickstart/index.md` (クイックガイド) のみを渡した状態で、事前説明なしに試合の記録（点数入力〜確定）を完了できるか。
- [ ] **文字サイズと配置**: 高齢の先生が老眼鏡なしでも、「面・小手・胴・突き」のボタンを押し間違えないUIサイズが確保されているか。

## 3. カオステスト (Chaos Engineering Drill)
試合進行中の想定外のトラブルに対する、人間の回復力を検証する。

- [ ] **Wi-Fi断**: 試合入力中に意図的にルーターの電源を落とす。エラー画面でフリーズせず、入力が継続でき、通信復旧後に同期されるか。
- [ ] **電源断（端末クラッシュ）**: 試合入力中に意図的にiPadを強制再起動する。
    - 現場の人間が速やかに「紙の記録」へ切り替えられるか。（クイックガイドの指示通りに動けるか）
    - 端末復旧後、紙の記録をもとにEvent Sourcingのリプレイ入力を行い、正しい状態に復元できるか。
