# Rule Testing Standard (ルールテスト基準)

ルールの正確性とリプレイの完全性を保証するために、すべての `RuleModule` に適用されるテスト基準を定義します。

## 1. 必須テストマトリクス (Required Test Matrix)
新しいルールを追加、または既存のルールをメジャーアップデート（バージョン更新）する際は、以下のテストが必須となります。

| テスト種別 | 目的 | 判定基準 |
| :--- | :--- | :--- |
| **Unit Test** | 単体ロジックの検証 | 特定の入力に対して期待通りの `RuleResult` を返すこと。 |
| **Replay Test** | 歴史的不変性の検証 | 過去の確定イベント履歴を再構築した際、勝敗結果が 1 bit も変化しないこと。 |
| **Property-based** | 不変条件の検証 | ランダムな入力の組み合わせ（1万回以上）でも、不変条件（例：スコアが負にならない等）を維持すること。 |
| **Chaos Test** | 異常系耐性の検証 | 不正な設定値や矛盾したイベントが注入されても、クラッシュ（Exception）しないこと。 |

## 2. カバレッジ基準 (Coverage Rule)
ルールエンジンのコアロジック（`analyzeHistory`, `decideResult`）に関しては、**C1カバレッジ（分岐網羅）100%** を原則とします。

## 3. リプレイ・回帰パイプライン (Replay Regression)
すべてのリリース前には、既存のすべての大会ゴールデンデータを対象としたリプレイ回帰テストを実行し、意図しない「歴史改変」が起きていないことを証明しなければなりません。
