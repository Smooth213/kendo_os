# Domain Glossary (ドメイン用語集)

エンジニアと競技運営者が共通認識を持つための言葉の定義です。本ドキュメントはシステムおよびAIの統一言語（Ubiquitous Language）として機能します。

### 🤺 競技・試合用語
- **有効打突 (Valid Strike):** 審判によって認められた一本。システム上は `isIppon: true` として記録される。
- **反則 (Hansoku):** 競技規則違反。累積2回で相手に一本が与えられる。
- **代表戦 (Representative Match):** 団体戦で決着がつかない場合に行われる決定戦。

### ⚙️ システム・統治用語 (Governance Vocabulary)
- **決定論的リプレイ (Deterministic Replay):** 同一のイベント履歴から常に同一の結果を再現すること。
- **Replay Drift (歴史改変 / 再生乖離):** システム改修により、過去のイベントログを入力とした際の結果（勝敗・スコア）が変化してしまう致命的なバグ。
- **Projection (射影):** 唯一の真実であるイベントストリームから、UI表示等のために導出・計算された読み取り専用のデータモデル（Read Model）。
- **補償イベント (Compensating Event):** 過去のデータを直接消す・上書きするのではなく、「取り消し（Undo）」という新たな事実を積み重ねて修正するイベント。
- **権威端末 (Authoritative Device):** データの最終決定権を持つ、記録係の端末。
- **Authoritative (権威ある / 最終裁定権):** データや判定の「絶対的な正解」を持つ主体。本システムでは常に「人間の入力（イベント）」がAuthoritativeである。
- **Fail-open (フェイルオープン / 継続許可):** ネットワーク等の一部機能が失敗しても、ローカル記録と試合進行を止めない障害時ポリシー。体育館運用で最優先される。
- **Fail-closed (フェイルクローズド / 進行停止):** データ整合性やルール評価が破綻した場合、誤った状態での進行を防ぐためシステムを明示的に停止させる障害時ポリシー。
