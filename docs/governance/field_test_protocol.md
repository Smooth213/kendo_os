# 体育館現場テスト要件 (Field Test Protocol)

**制定: マニュアルリンク統合フェーズ 完了時**

本ドキュメントは、Kendo Sync の「家電メーカー品質」を実際の体育館で検証するためのガバナンス要件である。アプリのリリース前には、以下のペルソナによる実地テストを必ずパスしなければならない。

## 1. テスト対象者 (Test Personas)
開発者によるテストは無効である。以下の人物をアサインすること。
- **高齢の審判 (60代以上)**: システムの自動判定に頼らず、主審としての権威（Human Override）を迷わず行使できるか。
- **初見の保護者 (観客)**: URLまたはQRコードを読み取るだけで、一切の説明なしに我が子の試合進行状況（Viewer）を把握できるか。
- **初見の高校生 (記録係)**: 3分間のクイックガイドを読むだけで、1試合の記録とUndo（取り消し）をノーミスで完遂できるか。

## 2. パニック・シミュレーション (Chaos Engineering for Gym)
以下の「劣悪な環境」を意図的に引き起こし、大会が継続できるかをテストする。
1. **ネットワーク切断テスト**: 試合中にWi-Fiルーターの電源を抜く。記録係がパニックにならず入力を継続し、復旧時にViewerが同期されるか。
2. **バッテリー枯渇・端末フリーズテスト**: 試合中にiPadを強制終了する。机上の「紙」へシームレスにフォールバックし、後からイベントを再入力（Replay）して歴史を結合できるか。
