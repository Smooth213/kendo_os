# 詰まり箇所分析レポート (Bottleneck Analysis Template)

無説明運用テスト（Step 10-2）において、ユーザーが操作に迷った、または失敗した事象を記録し、マニュアルやUIの改善に直結させるためのフォーマット。

## 発生事象 (Step 10-3)
- **対象ペルソナ**: [例: 中学生の記録係]
- **実施していた操作**: [例: 試合終了後の確定操作]
- **エラーの分類** (該当するものにチェック):
  - [ ] 読めなかった (文字が小さい、色が薄い、見つけられない)
  - [ ] 押し間違えた (ボタンが近い、アイコンの意味がわからない)
  - [ ] 用語不明 (マニュアル内の言葉の意味がわからなかった)

## 観察された行動
[例: 試合終了後、【確定】ボタンを「長押し」ではなく「タップ（短押し）」し続け、画面が変わらないと困惑していた。マニュアルの「長押し」という文字を見落としていた。]

## 改善アクション
- **UI側**: [例: タップされた時に「長押ししてください」とスナックバーで警告を出す]
- **マニュアル側**: [例: クイックガイドの「長押し」の文字を赤字の極太フォントに変更する]
