# 知識基盤の運用フリーズ宣言 (Manual Update Policy)

**制定: マニュアルリンク統合フェーズ 完了時**

本ドキュメントは、Kendo Syncの「コード」と「知識基盤（マニュアル）」の完全な同期を維持するための絶対的ルールである。

## 1. 唯一の真実 (Single Source of Truth)
本システムのドキュメントは、Markdownを唯一の真実とする。
Markdownの更新は、アプリ内のヘルプ、PDF、AI検索インデックスのすべてに波及する。これを手動で部分的に書き換えることは重大なガバナンス違反である。

## 2. マニュアル更新の絶対ルール (The Golden Rule)
**「UIやドメインルールの変更は、マニュアルの更新と同時に行われなければならない。」**

- 新しい画面を作成した場合は、必ず対応する `Manual ID` とマニュアル `.md` を作成すること。
- 既存の画面レイアウトを変更した場合は、`assets/manual_images/` の該当スクリーンショットを必ず差し替えること。
- ロジック（Event Sourcingやルールの根幹）を変更した場合は、マニュアル内の `Governance Notes` を必ず更新すること。

## 3. CIによる強制統治 (CI Enforcement)
プルリクエスト（PR）作成時、`doc_sync_validator.dart` が変更を自動検知する。
上記ルールに反して「コードだけ」が更新されたPRは、CI上で物理的にマージがブロック（FAIL）される。例外は一切認めない。
