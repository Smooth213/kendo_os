# 家電メーカー品質監査プロトコル (Appliance Quality Audit Protocol)

**「説明書だけで使える」を証明する最終テスト基準**

本システムを本番の大会へ投入する前に、以下の監査を必ず実施しなければならない。開発者による「口頭での補足説明」は、システムの敗北を意味する。

## 1. 初見ユーザーテスト (Step 10-1)
以下のペルソナを用意し、実際の体育館環境（照明、騒音、通信状態）でテストを行う。
- **高齢の審判 (60代以上)**: アプリのタイマーと自分の時計・感覚のズレがないか。
- **初見の保護者**: QRコードを読み取り、自分の子供の試合状況が把握できるか。
- **中学生・高校生の記録係**: 試合の進行スピードに合わせて入力が追いつくか。

## 2. 無説明運用テストの絶対条件 (Step 10-2)
テスト実施時、開発者およびシステム管理者は以下のルールを厳守すること。
1. **事前説明の禁止**: 「ここを押すと点数が入ります」等の口頭説明を一切してはならない。
2. **配布物の制限**: 被験者に渡して良いのは「端末」と「該当するマニュアル（クイックガイド）」のみとする。
3. **介入の禁止**: 被験者が操作に詰まっても、助け舟を出さず「マニュアルのどこを読んで、どう間違えたか」を観察する。

## 3. PDF印刷・オフライン閲覧監査 (Step 5-3, 5-4)
紙および電子媒体での完全性を担保するため、以下をテスト必須条件とする。
- **印刷テスト (Step 5-3)**: PDFを「A4・白黒設定」でプリントアウトし、体育館の薄暗い照明下で文字と画像（特に赤枠）が視認できるかを確認する。
- **オフライン閲覧確認 (Step 5-4)**: 端末（iPad, Androidタブレット）を「機内モード（Wi-Fiオフ）」にし、アプリ内の「？」ボタンからマニュアルが正常に開き、ローカル検索が機能することを確認する。

## 4. 体育館極限環境テスト (Step 7-4)
システムの最終認定として、実際の体育館にて以下の「最悪のシナリオ」を意図的に発生させ、大会運営が停止しないこと（Continuous）を証明しなければならない。
- **通信断 (Network Drop)**: 試合中にメインルーターの電源を抜く。記録係がエラー表示にパニックを起こさず、そのまま入力を継続できるか。
- **バッテリー低下・端末沈黙 (Battery Dead)**: 記録タブレットの電源を強制的に落とす。オペレーターが机上の「クイックガイド」を見て即座に紙運用へ切り替え、後でReplay（履歴再構築）できるか。
- **操作ミス (Human Error)**: 意図的に別の選手の得点ボタンを連打する。マニュアルの「困ったとき」導線から「Undo（取り消し）」に自力で到達し、即座に修正できるか。
