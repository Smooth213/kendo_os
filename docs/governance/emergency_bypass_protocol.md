# Emergency Governance Bypass (緊急ガバナンス・バイパス)

致命的なランタイムエラーにより競技進行が物理的に不可能な場合に限り、**人間（管理者）の明示的な操作**によってガバナンス・チェックを一時的にバイパスすることを許可する。

## 発動条件 (Bypass Conditions)
- ガバナンス・エンジンのデッドロックにより入力不能となった場合。
- 再生（Replay）不能な破損データが発生し、復旧が現場で不可能な場合。
- その他、人間の裁定により「システムよりも進行を優先すべき」と判断された場合。

## AIの制限
**AIは本バイパスを自ら発動することは決してできない。** バイパスの実行には必ず人間の物理的な操作または承認トークンを必要とする。

## 記録義務
バイパス実行時は、以下の内容を紙または別媒体に記録し、後に `Decision Ledger` へ登録しなければならない。
1. バイパス実行者名
2. 発生した障害の内容
3. バイパスによって許可された操作
