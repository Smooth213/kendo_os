# Documentation Domain Modeling (ドキュメント構造定義)

本ドキュメントは、Kendo Syncにおける「知識体系」を誰に、何を、どの深さで提供するかを定義する。

## 1. ドキュメント分類体系 (Documentation Taxonomy)
すべてのドキュメントは、以下のいずれかのカテゴリに分類されなければならない。
- **Operator**: 大会運営者・記録係向けの操作と運用手順
- **Viewer**: 観客・保護者向けの閲覧方法
- **Governance**: AI運用・Replay思想・監査哲学
- **Recovery**: 停電・通信断・端末停止時の緊急復旧手順
- **Admin**: システム管理者向けの設定とマスター管理
- **Architecture**: システムアーキテクチャと設計決定 (ADR)
- **FAQ**: よくある質問とトラブルシューティング
- **QuickGuide**: 現場向けの1〜2ページの即席手順

## 2. Audience Matrix (対象者別知識マトリクス)
それぞれの役割に対して、「必要な知識」と「システム不整合を防ぐために与えない知識（禁止知識）」を明確に分離する。

| 対象者 (Audience) | 必要知識 (Required Knowledge) | 禁止知識 (Restricted Knowledge) |
| :--- | :--- | :--- |
| **記録係 (Operator)** | 試合操作、Undo操作、紙運用への移行手順 | Rule Engine内部ロジック、ガバナンスアーキテクチャ |
| **観客 (Viewer)** | 画面の表示確認、更新タイミング | 障害復旧手順 (Recovery)、操作方法 |
| **大会管理者 (Admin)** | 大会作成、ルール設定、監査ログ(Audit)確認 | Rule DSLの直接記述、内部インフラ設定 |
| **監査官 (Governance)**| 全運用手順、Replay監視、アーキテクチャ | なし |

## 3. Manual Depth Level (知識の深度レベル)
ドキュメントの「深さ」を定義し、状況（平常時・緊急時）に応じて適切なレベルの情報を提供する。

| Level | 名称 | 内容 | 主な用途 |
| :---: | :--- | :--- | :--- |
| **L0** | Quick Guide | 現場での即読手順、最低限の操作 | 体育館の机上、試合前の3分間 |
| **L1** | 基本操作 | 画面の主要な目的と手順、注意点 | 平常時の操作学習、操作ヘルプ |
| **L2** | 詳細運用 | 状態遷移、誤操作時の振る舞い、エッジケース | トラブルシューティング、管理者確認 |
| **L3** | 障害対応 | 停電、ネットワーク断、カオスからの復旧手順 | 緊急時、大会を止めないための行動 |
| **L4** | Governance | なぜその設計か、Replay思想、監査の理由 | システム開発・改修、運用ルールの根拠確認 |
