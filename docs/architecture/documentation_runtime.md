# Documentation Runtime: The Operating System of Knowledge

**制定日: 2026年5月10日**
**最終更新: Phase 9 最終形態 統合**

Kendo Syncにおいて、ドキュメントは「付属品」ではない。コード、ガバナンス、そして運用知識が三位一体となり、初めて「競技運営OS」として機能する。本文書は、その統合状態を定義する。

## 1. 知識の循環構造 (Knowledge Lifecycle)
システムは以下の単一の真実（Markdown）を起点とした自動循環によって統治される。

1.  **Markdown Knowledge Base (The Source)**
    - すべての真実は `docs/manuals/` にプレーンテキストで刻まれる。
2.  **PDF / Quick Guide (Operational Projection)**
    - 現場の高齢者や初心者が、オフライン・過酷な環境で即座に動くための物理的な射影。
3.  **App Search / AI Search (Digital Runtime)**
    - アプリ内検索およびAIが、文脈とガバナンスを理解してユーザーを導く知能的実行環境。
4.  **Governance Runtime (CI/CD Enforcement)**
    - コードとドキュメントの同期を物理的に強制する、生存のための統治機構。

## 2. Documentation = Operating System
普通のアプリは「コード」が本体だが、Kendo Syncは以下の合計を本体とみなす。
- **Code**: 実行可能なロジック
- **Governance**: 整合性と歴史を守る規律
- **Documentation**: 迷いなき運用のための知識
- **Operational Knowledge**: 障害から復旧（Replay）させる知恵

## 3. 不変の運用原則 (Operational Invariants)
- **知識の不揮発性**: システムがダウンしても、クイックガイドと紙記録、そしてそこからのリプレイ手順により、大会の歴史は1bitも失われない。
- **AIとの共生**: ドキュメントは常にAI可読（Structured & Metadata tagged）であり、AIは常に「ドキュメント憲法」という法的枠組みの中で回答する。
