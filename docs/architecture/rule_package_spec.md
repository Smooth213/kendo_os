# Rule Package Spec (ルールパッケージ仕様)

将来的なルールの外部配信および DSL（独自言語）による定義に向けた、互換性と信頼の境界を定義します。

## 1. 互換性契約 (Compatibility Contract)
すべてのルールパッケージは、エンジン本体のバージョンとの互換性を明示しなければなりません。
- エンジンのメジャーバージョンが異なるパッケージの読み込みは、安全のためデフォルトでブロックされます。

## 2. サンドボックス・ポリシー (Sandbox Policy)
外部から供給されるルール、あるいは DSL によって動的に生成されるルールは、以下のサンドボックス制約下で実行されます。
- **No Side Effects:** システムのコア状態以外へのアクセスを完全に遮断します。
- **Resource Limit:** 無制限のループや再帰を禁止し、一定時間内に評価を完了させる制約を設けます。

## 3. 信頼境界 (Trust Boundary)
リモートで配布されるパッケージやユーザー定義の DSL は、検証（Validation）が完了するまで「信頼できない入力（Untrusted Input）」として扱われます。
- 実行前に、決定論的挙動を損なう命令（時刻取得、乱数生成等）が含まれていないか静的解析を行います。
