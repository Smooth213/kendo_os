# Replay Compatibility Policy (リプレイ互換性ポリシー)

過去の試合結果（イベント履歴）を再構築する際の一貫性を保証するためのルールを定義します。

## 1. 歴史的不変性の原則 (Historical Invariant)
一度確定した試合結果は、将来のルール変更によって影響を受けてはなりません。リプレイ時は、常に「イベント発行時のルールバージョン」を適用します。

## 2. バージョニング・ルール (Versioning Rules)
ルール変更の影響範囲に応じて、以下の通りバージョンを管理します。

- **Major Change:** 勝敗判定やスコア計算の意味論（Semantics）が変わる場合。
  - 例：一本勝負から三本勝負への変更、反則累積による一本付与ロジックの変更。
  - 対応：新しい `RuleModule` クラスを作成し、`ruleVersion` をインクリメントする。
- **Minor Change:** 表示名称やログの形式など、計算結果に影響しない変更。
  - 例：技の表示名の修正（"ド" -> "胴"）。
  - 対応：既存のモジュールを修正し、メタデータのみを更新する。

## 3. マイグレーションの禁止事項
- **In-place Mutation:** 既存のルールバージョンのロジックを直接書き換えて、過去の計算結果を変えることは厳禁です。
- **Silent Migration:** ユーザーの関与なしに過去のイベントデータの `ruleVersion` を自動的に書き換えてはなりません。

## 4. リプレイ監査手順 (Audit Procedure)
重要なリリース前には、`test/integration/replay_regression_test.dart` を実行し、過去のゴールデンデータ（確定済み試合）の計算結果が 1 bit も変化していないことを確認してください。
